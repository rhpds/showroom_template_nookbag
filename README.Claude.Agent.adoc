= Claude Agents for Red Hat Showroom
:toc:

== TL;DR - Quick Start

This repository has **8 AI agents** to help you create and convert Red Hat Showroom content.

**All agents use local files only** - no internet searches needed.

=== Quick Examples

**Convert existing content:**
----
Act as the content-converter agent and convert https://github.com/example/tutorial to Showroom format
----

**Review for Red Hat style:**
----
Act as the style-enforcer agent and check workshop/example/01-overview.adoc for Red Hat style compliance
----

**Create new content:**
----
Act as the technical-writer agent and create Module 1 for a container security workshop
----

== What Agents Do Automatically

All agents automatically:

* ✅ Read local prompt files from `.claude/prompts/`
* ✅ Follow Red Hat corporate style guidelines
* ✅ Reference local examples in `content/modules/ROOT/pages/`
* ✅ Apply sentence case headlines, product naming, inclusive language
* ❌ **Never** search the internet for style guides or standards

== Red Hat Style Quick Reference

Agents automatically enforce these rules:

[cols="1,2,2", options="header"]
|===
| Rule | ❌ Don't | ✅ Do

| Headlines
| "Getting Started With Containers"
| "Getting started with containers"

| Product names
| "The Red Hat OpenShift Platform"
| "Red Hat OpenShift"

| Numbers
| "Five ways to improve"
| "5 ways to improve"

| Commas
| "Security, scalability and performance"
| "Security, scalability, and performance"

| Inclusive language
| "whitelist", "master/slave"
| "allowlist", "primary/secondary"

| Vague terms
| "robust", "powerful", "leverage"
| Specific benefits/features
|===

== Troubleshooting

**Agent not following Red Hat style?**

Agents always read `.claude/prompts/redhat_style_guide_validation.txt` first. Check that this file exists.

**Conversion not working?**

The content-converter agent creates files in this order:
1. nav.adoc
2. 00-index.adoc
3. 01-overview.adoc
4. 02-details.adoc
5. Module files (03-*, 04-*, etc.)

**Agent searching the web?**

Only the researcher agent can use web search, and only as a last resort after checking local files. All other agents are prohibited from web access.
