= Module 2: AgnosticV Configuration (OCP Dedicated)

Add the following settings to your catalog item's `common.yaml`.

[source,yaml]
----
ocp4_workload_showroom_content_git_repo: https://github.com/YOUR-ORG/YOUR-SHOWROOM.git
ocp4_workload_showroom_content_git_repo_ref: main
ocp4_workload_showroom_content_antora_playbook: site.yml

# Terminal tab — showroom type gives a full /terminal/ tab (oc pre-configured)
ocp4_workload_showroom_terminal_type: showroom
ocp4_workload_showroom_wetty_image: "quay.io/rhpds/wetty:v3.0"

# E2E runtime automation
ocp4_workload_showroom_runtime_automation_enable: true
ocp4_workload_showroom_runtime_automation_image: "quay.io/rhpds/zt-runner:v2.4.2"

# ServiceAccount scope — cluster-admin required for dedicated clusters
# (solve.yml needs to create Namespaces, RoleBindings, etc.)
# Also valid for tenant labs if your playbooks need cluster-wide access.
ocp4_workload_runtime_automation_k8s_cluster_admin: true
ocp4_workload_runtime_automation_k8s_openshift_api_url: "{{ openshift_api_url }}"
ocp4_workload_runtime_automation_k8s_openshift_api_token: "{{ openshift_cluster_admin_token }}"

# Workloads — showroom + runner SA + E2E load test
agnosticd_workloads:
  - agnosticd.showroom.ocp4_workload_showroom
  - rhpds.ftl.ocp4_workload_runtime_automation_k8s
  - rhpds.ftl.ocp4_runtime_automation_load_test       # runs all solve+validate at provision time
----

== ServiceAccount Scope

`ocp4_workload_runtime_automation_k8s` creates a `zt-runner` ServiceAccount.

[cols="1,2"]
|===
| Setting | When to use

| Default (no flag) | SA gets `admin` RoleBinding in `student_ns` only. Only for tenant labs where solve.yml stays inside the student namespace.

| `ocp4_workload_runtime_automation_k8s_cluster_admin: true` | SA gets cluster-admin. *Required for dedicated clusters* (no `student_ns` — must create Namespaces itself). Also use for tenant labs if solve.yml needs cluster-scoped resources.
|===

NOTE: `student_ns` is empty for dedicated clusters — use a fixed namespace like `demo-project`
with `| default('demo-project', true) | trim` fallback in your playbooks.

== Load Test Role

`ocp4_runtime_automation_load_test` calls `/stream/solve` then `/stream/validate` for
every module at provision time and fails the provision if any validation fails.
This is what runs at Summit — students never see the Solve/Validate buttons.
