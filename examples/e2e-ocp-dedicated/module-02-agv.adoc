= Module 2: AgnosticV Configuration (OCP Dedicated)

Add the following settings to your catalog item's `common.yaml`.

[source,yaml]
----
ocp4_workload_showroom_content_git_repo: https://github.com/YOUR-ORG/YOUR-SHOWROOM.git
ocp4_workload_showroom_content_git_repo_ref: main
ocp4_workload_showroom_content_antora_playbook: site.yml

# Terminal tab — showroom type gives a full /terminal/ tab (oc pre-configured)
ocp4_workload_showroom_terminal_type: showroom
ocp4_workload_showroom_wetty_image: "quay.io/rhpds/wetty:v3.0"

# E2E runtime automation
ocp4_workload_showroom_runtime_automation_enable: true
ocp4_workload_showroom_runtime_automation_image: "quay.io/rhpds/zt-runner:v2.4.2"

# cluster-admin SA — required to create namespaces, RoleBindings, etc.
ocp4_workload_runtime_automation_k8s_cluster_admin: true
ocp4_workload_runtime_automation_k8s_openshift_api_url: "{{ openshift_api_url }}"
ocp4_workload_runtime_automation_k8s_openshift_api_token: "{{ openshift_cluster_admin_token }}"

# Workloads
agnosticd_workloads:
  - agnosticd.showroom.ocp4_workload_showroom
  - rhpds.ftl.ocp4_workload_runtime_automation_k8s
----

NOTE: `student_ns` is empty for dedicated clusters — use a fixed namespace with a
`| default('demo-project', true) | trim` fallback in your playbooks.
The bastion SSH credentials (`bastion_host`, `bastion_port`, `bastion_user`,
`bastion_password`) are injected automatically via showroom-userdata.
