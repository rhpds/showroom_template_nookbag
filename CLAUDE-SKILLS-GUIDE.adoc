## ðŸ’¡ Pro Tips

### For Workshop Creators

1. **Use real reference materials**: Better docs = better content
   - Red Hat product docs
   - Technical blog posts
   - Architecture guides

2. **Start with story**: Good scenario makes content relatable
   - ACME Corp is fine, but custom is better
   - Real business challenge = better engagement

3. **Provide UserInfo variables**: If you have a deployed RHDP environment
   - Login to demo.redhat.com or integration.demo.redhat.com
   - Go to "My services" â†’ Details â†’ Advanced settings
   - Copy/paste the output when asked
   - Gets you exact variable names for your environment

4. **Test commands**: Always verify in real environment before publishing

### For Demo Creators

1. **Quantify everything**: "6 weeks â†’ 5 minutes" beats "faster"

2. **Know your audience**:
   - Executives: Business value, ROI
   - Technical: Architecture, features
   - Sales Engineers: Competitive differentiators

3. **Practice the demo**: Use generated "Show" sections as your script

### For Both

1. **Use arguments for speed**:
   ```bash
   # Instead of answering directory every time
   /create-lab content/modules/ROOT/pages/ --continue 03-module-01.adoc
   ```

2. **Verify before committing**:
   ```bash
   /verify-content
   ```

3. **Fix issues incrementally**: Don't wait until all modules are done

## âš ï¸ Common Formatting Issues

### List Formatting (CRITICAL)

Lists without proper blank lines cause text to run together when rendered in Showroom.

**âŒ WRONG - Text runs together**:
```asciidoc
**Prerequisites:**
* OpenShift installed
* Admin access
Next section starts...
```

**âœ… CORRECT - Proper spacing**:
```asciidoc
**Prerequisites:**

* OpenShift installed
* Admin access

Next section starts...
```

**Required blank lines**:
1. Blank line after bold heading (`**Text:**`)
2. Blank line before first list item
3. Blank line after last list item

### Image Links (REQUIRED)

All images must be clickable to open full-size in new tab.

**âŒ WRONG - Not clickable**:
```asciidoc
image::diagram.png[My Diagram,width=700,title="Architecture"]
```

**âœ… CORRECT - Clickable image**:
```asciidoc
image::diagram.png[My Diagram,link=self,window=blank,width=700,title="Architecture"]
```

**With center alignment**:
```asciidoc
image::architecture.png[Architecture,link=self,window=blank,align="center",width=800,title="System Architecture"]
```

### Skill doesn't start

**Claude Code**:
```bash
# Make sure you're in a directory with .claude/ folder
ls .claude/skills/

```

Wait 5-10 minutes after PR is created. Integration environment syncs periodically.

### Generated content has issues

```bash

```

Review the report and fix critical issues.
